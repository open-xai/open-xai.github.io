Introduction

As machine learning models are increasingly being deployed in high-stakes settings such as finance and law, it becomes critical to ensure that individuals who are adversely affected by these model predictions are provided with a corresponding explanation. To this end, several algorithms have been proposed in eXplainable Artificial Intelligence (XAI) literature to generate explanations for affected individuals. However, the lack of high-quality benchmarks impedes the advancement of XAI. Here, OpenXAI supports the development of novel explanation methods and evaluation metrics, with a strong bent towards developing a public leaderboard to support systematic benchmarking of explanation methods.

OpenXAI is the first benchmark resource that unifies all four categories of evaluation metrics, namely disagreement, faithfulness, stability, and disparity metrics, in current XAI literature. It contains the most widely used XAI-ready structured datasets, explanation methods, benchmarks and leaderboards, and extensive Python programming functionality.

Datasets, trained models, explanation methods, and evaluation metrics in OpenXAI are XAI-ready, where preprocessed input features and pre-trained models are readily available for researchers and practitioners, such that they can directly use them for benchmarking explanation methods. The functionalities of OpenXAI are designed for easy integration in the XAI workflow. Finally, we develop an open-source software library that allows for efficient benchmarking of the quality of explanations with respect to several key properties.

Building trustworthy Explanation Methods for XAI
We envision OpenXAI to be the center between XAI researchers and practitioners. Researchers can propose, integrate, and benchmark new explanation methods into OpenXAI, and ML practitioners can integrate new evaluation metrics and obtain the leaderboard performance to use relevant explanation methods in their applications.

Installation

To install the OpenXAI package, clone OpenXAI's repo and install the package from the root directory using: 

pip install -e .

The installation of the OpenXAI package is hassle-free with minimum dependency on external packages.

Data Loaders

The data loaders in OpenXAI are lightweight. OpenXAI provides a collection of functionalities with easy-toa-use high-level APIs to benchmark explanations. As an example, to obtain the SynthGauss dataset from the data loader in OpenXAI, do as follows:

from openxai.dataloader import return_loaders
loader_train, loader_test = return_loaders(data_name= 'synthgauss', download=True)
inputs, labels = iter(loader_test).next()

Ecosystem of Datasets, Explainers, Tools, Evaluation Metrics, and Community Resources

OpenXAI includes numerous data functions that can be readily used with any explanation method. OpenXAI divides its programmatic ecosystem into four broad categories:

  • Dataloader and Processing: The dataset class in OpenXAI provides a base template of the dataset class for easier and faster incorporation of new datasets in the evaluation pipeline of OpenXAI. The dataloader in OpenXAI has been developed to output XAI-ready datasets, i.e., providing all intrinsic information of a give dataset including ground-truth explanations and distributions.
  • Explanation: The core component of OpenXAI is the explanation class which provides an abstract class to implement custom explanation methods for benchmarking. It incorporates a get_explanation method that returns explanations for a given set of input and model predictor.
  • Evaluation: OpenXAI includes an extensive list of evaluation metrics for quantifying the reliability of generated explanations. The evaluation class leverages generated explanations, ground-truth explanations, and other user-controlled parameters like top-k features.
